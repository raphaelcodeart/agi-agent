@AGENTS.md

Before modifying code:

1. Read the relevant documentation in /docs.
2. Inspect the existing implementation.
3. Identify impacted database models, API endpoints, tasks and frontend components.
4. Make the smallest coherent change.
5. Add or update tests.
6. Run backend tests, frontend type checks and linting.
7. Do not silently replace working architecture.
8. Do not fabricate Buffer API behavior.
9. Keep production and fake adapters strictly separated.
10. Update documentation when behavior changes.
